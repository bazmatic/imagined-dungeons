# Spawn Narration Design

**Date:** 2026-05-15
**Status:** Approved

## Problem

When a spawn trigger fires and agents appear in a location, the player currently sees only a static template string — `"Ash Zombie appeared"` — generated by `renderAgentSpawnedObserved`. There is no atmospheric description of *how* the agents arrived. This breaks immersion at a high-drama moment (zombies emerging from ash, a creature materialising from shadow, etc.).

The consequence engine does not currently run on `AgentSpawned` events because the spawn pass runs after all consequence passes in the tick pipeline.

## Goal

After a spawn fires, the player sees:
1. The existing mechanical line: `"Ash Zombie appeared"` (unchanged)
2. Immediately followed by a vivid, LLM-generated, **transient** narration describing the arrival — e.g. *"From the choking ash at the alley's far end, two blackened figures lurch upright, their hollow eyes fixing on you with cold hunger."*

The narration is transient — it does not update stored location or agent descriptions in the DB.

## Tick Pipeline Change

Current order:

```
Consequence pass (depth 0)
Consequence pass (depth 1)
Spawn pass  →  AgentSpawned events
Wake / sleep NPCs
```

New order:

```
Consequence pass (depth 0)
Consequence pass (depth 1)
Spawn pass  →  AgentSpawned events
Spawn narration pass  →  narration strings appended to witnessed[]   ← NEW
Wake / sleep NPCs
```

The spawn narration pass only runs when the spawn pass produced at least one `AgentSpawned` event. It does not feed back into any further consequence or spawn passes.

## `generateSpawnNarration` Function

**Location:** `src/core/spawning/narration.ts` (new file)

**Signature:**
```ts
export async function generateSpawnNarration(args: {
  readonly spawnEvents: readonly AgentSpawnedEvent[];
  readonly playerId: AgentId;
  readonly repo: Repository;
  readonly llm: LanguageModel | null;
}): Promise<readonly string[]>
```

**Behaviour:**

1. Groups `AgentSpawned` events by `locationId`, keeping only groups where `playerId` appears in the `witnesses` of at least one event in the group — i.e. the player was present when the spawn landed.
2. For each qualifying location group, fetches:
   - The location's `shortDescription` and `longDescription`
   - Each spawned agent's `label`, `shortDescription`, and `longDescription`
3. Makes one LLM call per location group using a focused schema: `{ narration: string }`.
4. Returns one narration string per location group.
5. If `llm` is null (LLM disabled or in tests), returns `[]` — the player still sees the `"X appeared"` line from the existing event rendering. No error.

**System prompt guidance:**
> You are a dungeon master narrating a tabletop RPG. Write a vivid, present-tense description of these creatures arriving in this location. Two to three sentences. Do not reference game mechanics or stats.

**User prompt content:**
- Location label and description
- List of spawned agents with their labels and descriptions

## Output Path

The narration strings returned by `generateSpawnNarration` are appended directly to `witnessed[]` in `runTick`, immediately after the spawn pass:

```ts
const spawnNarrations = await generateSpawnNarration({ spawnEvents, playerId, repo, llm });
witnessed.push(...spawnNarrations);
```

No new event kinds are introduced. No dispatch. No DB writes.

## Player Experience

For a player in Ash Lane when the LlmJudgement trigger fires:

```
> shout

You shout into the darkness.

Ash Zombie appeared.
From the choking ash at the alley's far end, two blackened figures lurch upright,
their hollow eyes fixing on you with cold hunger.
```

## Edge Cases

| Scenario | Behaviour |
|---|---|
| `llm` is null | Only `"X appeared"` shown; no error |
| Multiple agent types, same location | Single narration call describes them together |
| Spawns at two different locations in one tick | One narration call per location; each scoped to that location only |
| Spawn trigger fires but no player is present | `generateSpawnNarration` skips that location group (player not in witnesses); no narration generated; player only sees the `"X appeared"` line if they are a witness, otherwise nothing |

## Out of Scope

- Updating stored location or agent descriptions (use the consequence engine for that separately)
- Narrating despawn / agent death in the same fashion (separate feature)
- The "Save Seed to disk" persistence question raised earlier in the same session — separate concern, not yet scoped
